# 📝 Lesson 2: The Comment

Code is for computers, but **Comments** are for humans. When you are building a complex robot, you will forget why you wrote a certain line of code. Comments save your future self.

## 1. The 5-Year-Old Spark ⚡
Imagine you are building a Lego set and you leave a sticky note on a piece that says "This part is for the left wing." The Lego set doesn't care about the note, but it helps *you* remember what you're doing.

## 2. The Deep Dive 🤿
In Python, any line starting with a `#` (hash/pound symbol) is ignored by the computer.
* **Single-line comments:** Use `#`.
* **The "Why", not the "What":** Don't use a comment to say `print("Hi") # prints hi`. Use it to explain *why* you are doing it: `print("Hi") # Greeting the user to start the UI`.

## 3. Side-by-Side Mirror 🪞

**Python:**
```python
# This is a comment
print("Focus on the code!")
```

**JavaScript:**
```javascript
// This is a comment
console.log("Focus on the code!");
```
